login
﻿

POST
Admin login with fullName and password
//api/auth/admin/login
API Request Description
This endpoint allows an admin user to log in.

Request Body
fullName (text, required): The full name of the admin user.
password (text, required): The password for the admin user.
Response
The response will be in JSON format and will include the following fields:

user (object): An object containing user details including id, full name, email, phone number, role, avatar URL, email verification status, creation and update timestamps.
accessToken (string): The access token for the logged-in admin user.
expiresIn (string): The expiration time for the access token.
JSON Schema for Response
JSON
{
  "type": "object",
  "properties": {
    "user": {
      "type": "object",
      "properties": {
        "id": {"type": "string"},
        "fullName": {"type": "string"},
        "email": {"type": "string"},
        "phoneNumber": {"type": ["string", "null"]},
        "role": {"type": "string"},
        "avatarUrl": {"type": ["string", "null"]},
        "isEmailVerified": {"type": "boolean"},
        "createdAt": {"type": "string"},
        "updatedAt": {"type": "string"}
      }
    },
    "accessToken": {"type": "string"},
    "expiresIn": {"type": "string"}
  }
}
This endpoint allows administrators to log in.

Request Body
fullName (string) - The full name of the admin user.
password (string) - The password for the admin user.
Response
Upon successful login, the response will include:

user (object) - An object containing user details including id, full name, email, phone number, role, avatar URL, email verification status, creation and update timestamps.
accessToken (string) - The access token for the authenticated session.
expiresIn (string) - The expiration time for the access token.
Example Response
JSON
{
    "user": {
        "id": "",
        "fullName": "",
        "email": "",
        "phoneNumber": null,
        "role": "",
        "avatarUrl": null,
        "isEmailVerified": true,
        "createdAt": "",
        "updatedAt": ""
    },
    "accessToken": "",
    "expiresIn": ""
}

POST
Create a new product (admin only)
//api/products
API POST /api/products
This endpoint allows the client to create a new product by providing the necessary details in the request body.

Request Body
name (string) - The name of the product.
description (string) - A brief description of the product.
price (number) - The price of the product.
stockQuantity (number) - The quantity of the product in stock.
